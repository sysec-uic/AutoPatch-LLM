AutoPatch-LLM

Automated Program Repair for Vulnerabilities using Large Language Models

A project by the Systems Security Lab (SySeC) at the University of Illinois Chicago (UIC)

About The Project

AutoPatch-LLM leverages the power of Large Language Models (LLMs), such as GPT-4, to automatically generate patches for security vulnerabilities found in C/C++ code. Given a description of the vulnerability (like a CVE report) and the vulnerable code snippet, AutoPatch-LLM aims to produce a correct patch to remediate the issue.

This tool is designed to assist developers and security researchers in the often complex and time-consuming task of vulnerability patching.

Key Features

  • Automated patch generation for C/C++ vulnerabilities.
  • Utilizes state-of-the-art Large Language Models.
  • Processes vulnerability descriptions and code context.
  • Aims to reduce manual effort in patch development.
  • Research-driven approach developed at UIC SySeC Lab.

Getting Started

To get started with AutoPatch-LLM, follow these general steps (refer to the repository's README for detailed instructions):

  1. Clone the repository:
    git clone https://github.com/sysec-uic/AutoPatch-LLM.git
    cd AutoPatch-LLM
  2. Set up the environment:
    Install Python dependencies (likely using pip and a requirements.txt file).
    pip install -r requirements.txt (Example command)
  3. Configure API Keys:
    You will likely need API keys for the LLM service being used (e.g., OpenAI). Follow the project's instructions for configuration.
  4. Run the tool:
    Execute the main Python script, providing necessary inputs like vulnerability information and code file paths. Example (check actual usage in README):
    python autopatch_llm.py --vuln-desc "description.txt" --code "vulnerable_code.c"

Please consult the project's README for specific commands, options, and prerequisites.
